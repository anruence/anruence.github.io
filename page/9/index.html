Dependency InjectionпјҢеҚівҖңдҫқиө–жіЁе…ҘвҖқпјҡз»„д»¶д№Ӣй—ҙдҫқиө–е…ізі»з”ұе®№еҷЁеңЁиҝҗиЎҢжңҹеҶіе®ҡпјҢеҪўиұЎзҡ„иҜҙпјҢеҚіз”ұе®№еҷЁеҠЁжҖҒзҡ„е°ҶжҹҗдёӘдҫқиө–е…ізі»жіЁе…ҘеҲ°з»„д»¶д№ӢдёӯгҖӮдҫқиө–жіЁе…Ҙзҡ„зӣ®зҡ„е№¶йқһдёәиҪҜд»¶зі»з»ҹеёҰжқҘжӣҙеӨҡеҠҹиғҪпјҢиҖҢжҳҜдёәдәҶжҸҗеҚҮз»„д»¶йҮҚз”Ёзҡ„йў‘зҺҮпјҢе№¶дёәзі»з»ҹжҗӯе»әдёҖдёӘзҒөжҙ»гҖҒеҸҜжү©еұ•зҡ„е№іеҸ°гҖӮйҖҡиҝҮдҫқиө–жіЁе…ҘжңәеҲ¶пјҢжҲ‘д»¬еҸӘйңҖиҰҒйҖҡиҝҮз®ҖеҚ•зҡ„й…ҚзҪ®пјҢиҖҢж— йңҖд»»дҪ•д»Јз Ғе°ұеҸҜжҢҮе®ҡзӣ®ж ҮйңҖиҰҒзҡ„иө„жәҗпјҢе®ҢжҲҗиҮӘиә«зҡ„дёҡеҠЎйҖ»иҫ‘пјҢиҖҢдёҚйңҖиҰҒе…іеҝғе…·дҪ“зҡ„иө„жәҗжқҘиҮӘдҪ•еӨ„пјҢз”ұи°Ғе®һзҺ°гҖӮ
зҗҶи§ЈDIзҡ„е…ій”®жҳҜпјҡвҖңи°Ғдҫқиө–и°ҒпјҢдёәд»Җд№ҲйңҖиҰҒдҫқиө–пјҢи°ҒжіЁе…Ҙи°ҒпјҢжіЁе…ҘдәҶд»Җд№ҲвҖқпјҢйӮЈжҲ‘д»¬жқҘж·ұе…ҘеҲҶжһҗдёҖдёӢпјҡ

  • и°Ғдҫқиө–дәҺи°ҒпјҡеҪ“з„¶жҳҜеә”з”ЁзЁӢеәҸдҫқиө–дәҺIoCе®№еҷЁпјӣ
  • дёәд»Җд№ҲйңҖиҰҒдҫқиө–пјҡеә”з”ЁзЁӢеәҸйңҖиҰҒIoCе®№еҷЁжқҘжҸҗдҫӣеҜ№иұЎйңҖиҰҒзҡ„еӨ–йғЁиө„жәҗпјӣ
  • и°ҒжіЁе…Ҙи°ҒпјҡеҫҲжҳҺжҳҫжҳҜIoCе®№еҷЁжіЁе…Ҙеә”з”ЁзЁӢеәҸжҹҗдёӘеҜ№иұЎпјҢеә”з”ЁзЁӢеәҸдҫқиө–зҡ„еҜ№иұЎпјӣ
  • жіЁе…ҘдәҶд»Җд№Ҳпјҡе°ұжҳҜжіЁе…ҘжҹҗдёӘеҜ№иұЎжүҖйңҖиҰҒзҡ„еӨ–йғЁиө„жәҗпјҲеҢ…жӢ¬еҜ№иұЎгҖҒиө„жәҗгҖҒеёёйҮҸж•°жҚ®пјүгҖӮ

IoCе’ҢDIз”ұд»Җд№Ҳе…ізі»е‘ўпјҹе…¶е®һе®ғд»¬жҳҜеҗҢдёҖдёӘжҰӮеҝөзҡ„дёҚеҗҢи§’еәҰжҸҸиҝ°пјҢз”ұдәҺжҺ§еҲ¶еҸҚиҪ¬жҰӮеҝөжҜ”иҫғеҗ«зіҠпјҲеҸҜиғҪеҸӘжҳҜзҗҶи§Јдёәе®№еҷЁжҺ§еҲ¶еҜ№иұЎиҝҷдёҖдёӘеұӮйқўпјҢеҫҲйҡҫи®©дәәжғіеҲ°и°ҒжқҘз»ҙжҠӨеҜ№иұЎе…ізі»пјүпјҢжүҖд»Ҙ2004е№ҙеӨ§еёҲзә§дәәзү©Martin FowlerеҸҲз»ҷеҮәдәҶдёҖдёӘж–°зҡ„еҗҚеӯ—пјҡвҖңдҫқиө–жіЁе…ҘвҖқпјҢзӣёеҜ№IoC иҖҢиЁҖпјҢвҖңдҫқиө–жіЁе…ҘвҖқжҳҺзЎ®жҸҸиҝ°дәҶвҖңиў«жіЁе…ҘеҜ№иұЎдҫқиө–IoCе®№еҷЁй…ҚзҪ®дҫқиө–еҜ№иұЎвҖқгҖӮ

зңӢиҝҮеҫҲеӨҡеҜ№Springзҡ„IocзҗҶи§Јзҡ„ж–Үз« пјҢеҘҪеӨҡдәәеҜ№Iocе’ҢDIзҡ„и§ЈйҮҠйғҪжҷҰж¶©йҡҫжҮӮпјҢеҸҚжӯЈе°ұжҳҜдёҖз§ҚиҜҙдёҚжё…пјҢйҒ“дёҚжҳҺзҡ„ж„ҹи§үпјҢиҜ»е®Ңд№ӢеҗҺдҫқз„¶жҳҜдёҖеӨҙйӣҫж°ҙпјҢж„ҹи§үе°ұжҳҜејҖж¶ӣиҝҷдҪҚжҠҖжңҜзүӣдәәеҶҷеҫ—зү№еҲ«йҖҡдҝ—жҳ“жҮӮпјҢд»–жё…жҘҡең°и§ЈйҮҠдәҶIoC(жҺ§еҲ¶еҸҚиҪ¬) е’ҢDI(дҫқиө–жіЁе…Ҙ)дёӯзҡ„жҜҸдёҖдёӘеӯ—пјҢиҜ»е®Ңд№ӢеҗҺз»ҷдәәдёҖз§ҚиұҒз„¶ејҖжң—зҡ„ж„ҹи§үгҖӮжҲ‘зӣёдҝЎеҜ№дәҺеҲқеӯҰSpringжЎҶжһ¶зҡ„дәәеҜ№Iocзҡ„зҗҶи§Јеә”иҜҘжҳҜжңүеҫҲеӨ§её®еҠ©зҡ„гҖӮ

(donвҖҳt call us, weвҖҳll call you)вҖқиҝҷжҳҜи‘—еҗҚзҡ„еҘҪиҺұеқһеҺҹеҲҷгҖӮеңЁеҘҪиҺұеқһпјҢжҠҠз®ҖеҺҶйҖ’дәӨз»ҷжј”иүәе…¬еҸёеҗҺе°ұеҸӘжңүеӣһе®¶зӯүеҫ…гҖӮз”ұжј”иүәе…¬еҸёеҜ№ж•ҙдёӘеЁұд№җйЎ№зҡ„е®Ңе…ЁжҺ§еҲ¶пјҢжј”е‘ҳеҸӘиғҪиў«еҠЁејҸзҡ„жҺҘеҸ—е…¬еҸёзҡ„е·®дҪҝ,еңЁйңҖиҰҒзҡ„зҺҜиҠӮдёӯпјҢе®ҢжҲҗиҮӘе·ұзҡ„жј”еҮәгҖӮиҝҷдёҖзӮ№е®ҢзҫҺзҡ„жҸҗзҺ°дәҶеңЁIOCиә«дёҠпјҢIOCжүҖжіЁйҮҚзҡ„жҳҜи®ҫи®ЎжҖқжғідёҠпјҢд»ҺдёҖдёӘеёёи§„зҡ„еҲӣе»әеҜ№иұЎзҡ„еҒҡжі•пјҢеҚіnewдёҖдёӘеҜ№иұЎпјҢиҪ¬еҸҳжҲҗеҗ‘IOCе®№еҷЁйҖ’дәӨвҖқз®ҖеҺҶвҖңпјҢиў«еҠЁзҡ„зӯүеҫ…IOCе®№еҷЁиҝ”еӣһиө„жәҗз»ҷдҪ гҖӮжҺ§еҲ¶еҸҚиҪ¬еҚіжҢҮзҡ„жҳҜвҖқжј”иүәе…¬еҸёжҺ§еҲ¶жј”е‘ҳвҖңпјҢиҖҢиҜҙеҲ°дҫқиө–пјҢеҲҷжҳҜвҖңжј”е‘ҳйңҖиҰҒе…¬еҸёж··йҘӯвҖқпјҢжҲ‘д»¬жүҖйңҖжұӮзҡ„еҜ№иұЎпјҢйңҖиҰҒдҫқиө–е®№еҷЁжқҘиҺ·еҫ—пјҢиҝҷдёӘиҝҮзЁӢеҚіжҳҜдҫқиө–жіЁе…ҘгҖӮжң¬иҙЁдёҠIOCе’ҢDIжҳҜеҗҢдёҖжҖқжғідёӢдёҚеҗҢз»ҙеәҰзҡ„иЎЁзҺ°гҖӮ

According to the HTTP/1.1 Spec:

In other words, POST is used to create.

That is, PUT is used to create or update.

So, which one should be used to create a resource? Or one needs to support both?

Better is to choose between PUT and POST based on idempotence of the action.

PUT implies putting a resource - completely replacing whatever is available at the given URL with a different thing. By definition, a PUT is idempotent. Do it as many times as you like, and the result is the same. x=5 is idempotent. You can PUT a resource whether it previously exists, or not (eg, to Create, or to Update)!

POST updates a resource, adds a subsidiary resource, or causes a change. A POST is not idempotent, in the way that x++ is not idempotent.

By this argument, PUT is for creating when you know the URL of the thing you will create. POST can be used to create when you know the URL of the вҖңfactoryвҖқ or manager for the category of things you want to create.

POST /expense-report

еҸ‘иЎЁдәҺ 2018-01-12 жӣҙж–°дәҺ 2020-10-26 еҲҶзұ»дәҺ HTTP
жң¬ж–Үеӯ—ж•°пјҡ 6.1k йҳ…иҜ»ж—¶й•ҝ ≈ 6 еҲҶй’ҹ

з®Җд»Ӣ

еҮәдәҺе®үе…ЁеҺҹеӣ пјҢжөҸи§ҲеҷЁйҷҗеҲ¶д»Һи„ҡжң¬еҶ…еҸ‘иө·зҡ„и·ЁжәҗHTTPиҜ·жұӮгҖӮ дҫӢеҰӮпјҢXMLHttpRequestе’ҢFetch APIйҒөеҫӘеҗҢжәҗзӯ–з•ҘгҖӮ иҝҷж„Ҹе‘ізқҖдҪҝз”ЁиҝҷдәӣAPIзҡ„Webеә”з”ЁзЁӢеәҸеҸӘиғҪд»ҺеҠ иҪҪеә”з”ЁзЁӢеәҸзҡ„еҗҢдёҖдёӘеҹҹиҜ·жұӮHTTPиө„жәҗпјҢйҷӨйқһдҪҝз”ЁCORSеӨҙж–Үд»¶гҖӮ
и·Ёеҹҹиө„жәҗе…ұдә«пјҲ CORS пјүжңәеҲ¶е…Ғи®ё Web еә”з”ЁжңҚеҠЎеҷЁиҝӣиЎҢи·Ёеҹҹи®ҝй—®жҺ§еҲ¶пјҢд»ҺиҖҢдҪҝи·Ёеҹҹж•°жҚ®дј иҫ“еҫ—д»Ҙе®үе…ЁиҝӣиЎҢгҖӮжөҸи§ҲеҷЁж”ҜжҢҒеңЁ API е®№еҷЁдёӯпјҲдҫӢеҰӮ XMLHttpRequest жҲ– Fetch пјүдҪҝз”Ё CORSпјҢд»ҘйҷҚдҪҺи·Ёеҹҹ HTTP иҜ·жұӮжүҖеёҰжқҘзҡ„йЈҺйҷ©гҖӮ
CORSйңҖиҰҒжөҸи§ҲеҷЁе’ҢжңҚеҠЎеҷЁеҗҢж—¶ж”ҜжҢҒгҖӮзӣ®еүҚпјҢжүҖжңүжөҸи§ҲеҷЁйғҪж”ҜжҢҒиҜҘеҠҹиғҪпјҢIEжөҸи§ҲеҷЁдёҚиғҪдҪҺдәҺIE10гҖӮ
ж•ҙдёӘCORSйҖҡдҝЎиҝҮзЁӢпјҢйғҪжҳҜжөҸи§ҲеҷЁиҮӘеҠЁе®ҢжҲҗпјҢдёҚйңҖиҰҒз”ЁжҲ·еҸӮдёҺгҖӮеҜ№дәҺејҖеҸ‘иҖ…жқҘиҜҙпјҢCORSйҖҡдҝЎдёҺеҗҢжәҗзҡ„AJAXйҖҡдҝЎжІЎжңүе·®еҲ«пјҢд»Јз Ғе®Ңе…ЁдёҖж ·гҖӮжөҸи§ҲеҷЁдёҖж—ҰеҸ‘зҺ°AJAXиҜ·жұӮи·ЁжәҗпјҢе°ұдјҡиҮӘеҠЁж·»еҠ дёҖдәӣйҷ„еҠ зҡ„еӨҙдҝЎжҒҜпјҢжңүж—¶иҝҳдјҡеӨҡеҮәдёҖж¬Ўйҷ„еҠ зҡ„иҜ·жұӮпјҢдҪҶз”ЁжҲ·дёҚдјҡжңүж„ҹи§үгҖӮ
еӣ жӯӨпјҢе®һзҺ°CORSйҖҡдҝЎзҡ„е…ій”®жҳҜжңҚеҠЎеҷЁгҖӮеҸӘиҰҒжңҚеҠЎеҷЁе®һзҺ°дәҶCORSжҺҘеҸЈпјҢе°ұеҸҜд»Ҙи·ЁжәҗйҖҡдҝЎгҖӮ
еҪ“дёҖдёӘиө„жәҗд»ҺдёҺиҜҘиө„жәҗжң¬иә«жүҖеңЁзҡ„жңҚеҠЎеҷЁдёҚеҗҢзҡ„еҹҹжҲ–з«ҜеҸЈиҜ·жұӮдёҖдёӘиө„жәҗж—¶пјҢиө„жәҗдјҡеҸ‘иө·дёҖдёӘи·Ёеҹҹ HTTP иҜ·жұӮгҖӮ

з®ҖеҚ•иҜ·жұӮ

жөҒзЁӢ

жҹҗдәӣиҜ·жұӮдёҚдјҡи§ҰеҸ‘ CORS йў„жЈҖиҜ·жұӮгҖӮжң¬ж–Үз§°иҝҷж ·зҡ„иҜ·жұӮдёәвҖңз®ҖеҚ•иҜ·жұӮвҖқпјҢиҜ·жіЁж„ҸпјҢиҜҘжңҜиҜӯе№¶дёҚеұһдәҺ Fetch пјҲе…¶дёӯе®ҡд№үдәҶ CORSпјүи§„иҢғгҖӮиӢҘиҜ·жұӮж»Ўи¶іжүҖжңүдёӢиҝ°жқЎд»¶пјҢеҲҷиҜҘиҜ·жұӮеҸҜи§ҶдёәвҖңз®ҖеҚ•иҜ·жұӮвҖқпјҡ

еҸҰеӨ–Fetch и§„иҢғе®ҡд№үдәҶеҜ№ CORS е®үе…Ёзҡ„йҰ–йғЁеӯ—ж®өйӣҶеҗҲпјҢдёҚеҫ—дәәдёәи®ҫзҪ®иҜҘйӣҶеҗҲд№ӢеӨ–зҡ„е…¶д»–йҰ–йғЁеӯ—ж®өгҖӮиҜҘйӣҶеҗҲдёәпјҡ

еҜ№дәҺз®ҖеҚ•иҜ·жұӮпјҢжөҸи§ҲеҷЁзӣҙжҺҘеҸ‘еҮәCORSиҜ·жұӮгҖӮе…·дҪ“жқҘиҜҙпјҢе°ұжҳҜеңЁеӨҙдҝЎжҒҜд№ӢдёӯпјҢеўһеҠ дёҖдёӘOriginеӯ—ж®өгҖӮ

дёӢйқўжҳҜдёҖдёӘдҫӢеӯҗпјҢжөҸи§ҲеҷЁеҸ‘зҺ°иҝҷж¬Ўи·ЁжәҗAJAXиҜ·жұӮжҳҜз®ҖеҚ•иҜ·жұӮпјҢе°ұиҮӘеҠЁеңЁеӨҙдҝЎжҒҜд№ӢдёӯпјҢж·»еҠ дёҖдёӘOriginеӯ—ж®өгҖӮ

дёҠйқўзҡ„еӨҙдҝЎжҒҜдёӯпјҢOriginеӯ—ж®өз”ЁжқҘиҜҙжҳҺпјҢжң¬ж¬ЎиҜ·жұӮжқҘиҮӘе“ӘдёӘжәҗпјҲеҚҸи®® + еҹҹеҗҚ + з«ҜеҸЈпјүгҖӮжңҚеҠЎеҷЁж №жҚ®иҝҷдёӘеҖјпјҢеҶіе®ҡжҳҜеҗҰеҗҢж„Ҹиҝҷж¬ЎиҜ·жұӮгҖӮ

еҰӮжһңOriginжҢҮе®ҡзҡ„жәҗпјҢдёҚеңЁи®ёеҸҜиҢғеӣҙеҶ…пјҢжңҚеҠЎеҷЁдјҡиҝ”еӣһдёҖдёӘжӯЈеёёзҡ„HTTPеӣһеә”гҖӮжөҸи§ҲеҷЁеҸ‘зҺ°пјҢиҝҷдёӘеӣһеә”зҡ„еӨҙдҝЎжҒҜжІЎжңүеҢ…еҗ«Access-Control-Allow-Originеӯ—ж®өпјҲиҜҰи§ҒдёӢж–ҮпјүпјҢе°ұзҹҘйҒ“еҮәй”ҷдәҶпјҢд»ҺиҖҢжҠӣеҮәдёҖдёӘй”ҷиҜҜпјҢиў«XMLHttpRequestзҡ„onerrorеӣһи°ғеҮҪж•°жҚ•иҺ·гҖӮжіЁж„ҸпјҢиҝҷз§Қй”ҷиҜҜж— жі•йҖҡиҝҮзҠ¶жҖҒз ҒиҜҶеҲ«пјҢеӣ дёәHTTPеӣһеә”зҡ„зҠ¶жҖҒз ҒжңүеҸҜиғҪжҳҜ200гҖӮ

еҰӮжһңOriginжҢҮе®ҡзҡ„еҹҹеҗҚеңЁи®ёеҸҜиҢғеӣҙеҶ…пјҢжңҚеҠЎеҷЁиҝ”еӣһзҡ„е“Қеә”пјҢдјҡеӨҡеҮәеҮ дёӘеӨҙдҝЎжҒҜеӯ—ж®өгҖӮ

дёҠйқўзҡ„еӨҙдҝЎжҒҜд№ӢдёӯпјҢжңүдёүдёӘдёҺCORSиҜ·жұӮзӣёе…ізҡ„еӯ—ж®өпјҢйғҪд»ҘAccess-Control-ејҖеӨҙгҖӮ

пјҲ1пјүAccess-Control-Allow-Origin

иҜҘеӯ—ж®өжҳҜеҝ…йЎ»зҡ„гҖӮе®ғзҡ„еҖјиҰҒд№ҲжҳҜиҜ·жұӮж—¶Originеӯ—ж®өзҡ„еҖјпјҢиҰҒд№ҲжҳҜдёҖдёӘ*пјҢиЎЁзӨәжҺҘеҸ—д»»ж„ҸеҹҹеҗҚзҡ„иҜ·жұӮгҖӮ

пјҲ2пјүAccess-Control-Allow-Credentials

иҜҘеӯ—ж®өеҸҜйҖүгҖӮе®ғзҡ„еҖјжҳҜдёҖдёӘеёғе°”еҖјпјҢиЎЁзӨәжҳҜеҗҰе…Ғи®ёеҸ‘йҖҒCookieгҖӮй»ҳи®Өжғ…еҶөдёӢпјҢCookieдёҚеҢ…жӢ¬еңЁCORSиҜ·жұӮд№ӢдёӯгҖӮи®ҫдёәtrueпјҢеҚіиЎЁзӨәжңҚеҠЎеҷЁжҳҺзЎ®и®ёеҸҜпјҢCookieеҸҜд»ҘеҢ…еҗ«еңЁиҜ·жұӮдёӯпјҢдёҖиө·еҸ‘з»ҷжңҚеҠЎеҷЁгҖӮиҝҷдёӘеҖјд№ҹеҸӘиғҪи®ҫдёәtrueпјҢеҰӮжһңжңҚеҠЎеҷЁдёҚиҰҒжөҸи§ҲеҷЁеҸ‘йҖҒCookieпјҢеҲ йҷӨиҜҘеӯ—ж®өеҚіеҸҜгҖӮ

пјҲ3пјүAccess-Control-Expose-Headers

иҜҘеӯ—ж®өеҸҜйҖүгҖӮCORSиҜ·жұӮж—¶пјҢXMLHttpRequestеҜ№иұЎзҡ„getResponseHeader()ж–№жі•еҸӘиғҪжӢҝеҲ°6дёӘеҹәжң¬еӯ—ж®өпјҡCache-ControlгҖҒContent-LanguageгҖҒContent-TypeгҖҒExpiresгҖҒLast-ModifiedгҖҒPragmaгҖӮеҰӮжһңжғіжӢҝеҲ°е…¶д»–еӯ—ж®өпјҢе°ұеҝ…йЎ»еңЁAccess-Control-Expose-HeadersйҮҢйқўжҢҮе®ҡгҖӮдёҠйқўзҡ„дҫӢеӯҗжҢҮе®ҡпјҢgetResponseHeader(вҖҳFooBarвҖҷ)еҸҜд»Ҙиҝ”еӣһFooBarеӯ—ж®өзҡ„еҖјгҖӮ

дёҠйқўиҜҙеҲ°пјҢCORSиҜ·жұӮй»ҳи®ӨдёҚеҸ‘йҖҒCookieе’ҢHTTPи®ӨиҜҒдҝЎжҒҜгҖӮеҰӮжһңиҰҒжҠҠCookieеҸ‘еҲ°жңҚеҠЎеҷЁпјҢдёҖж–№йқўиҰҒжңҚеҠЎеҷЁеҗҢж„ҸпјҢжҢҮе®ҡAccess-Control-Allow-Credentialsеӯ—ж®өгҖӮ

Access-Control-Allow-Credentials: true
еҸҰдёҖж–№йқўпјҢејҖеҸ‘иҖ…еҝ…йЎ»еңЁAJAXиҜ·жұӮдёӯжү“ејҖwithCredentialsеұһжҖ§гҖӮ

var xhr = new XMLHttpRequest();
xhr.withCredentials = true;
еҗҰеҲҷпјҢеҚідҪҝжңҚеҠЎеҷЁеҗҢж„ҸеҸ‘йҖҒCookieпјҢжөҸи§ҲеҷЁд№ҹдёҚдјҡеҸ‘йҖҒгҖӮжҲ–иҖ…пјҢжңҚеҠЎеҷЁиҰҒжұӮи®ҫзҪ®CookieпјҢжөҸи§ҲеҷЁд№ҹдёҚдјҡеӨ„зҗҶгҖӮ

дҪҶжҳҜпјҢеҰӮжһңзңҒз•ҘwithCredentialsи®ҫзҪ®пјҢжңүзҡ„жөҸи§ҲеҷЁиҝҳжҳҜдјҡдёҖиө·еҸ‘йҖҒCookieгҖӮиҝҷж—¶пјҢеҸҜд»ҘжҳҫејҸе…ій—ӯwithCredentialsгҖӮ

xhr.withCredentials = false;
йңҖиҰҒжіЁж„Ҹзҡ„жҳҜпјҢеҰӮжһңиҰҒеҸ‘йҖҒCookieпјҢAccess-Control-Allow-Originе°ұдёҚиғҪи®ҫдёәжҳҹеҸ·пјҢеҝ…йЎ»жҢҮе®ҡжҳҺзЎ®зҡ„гҖҒдёҺиҜ·жұӮзҪ‘йЎөдёҖиҮҙзҡ„еҹҹеҗҚгҖӮеҗҢж—¶пјҢCookieдҫқз„¶йҒөеҫӘеҗҢжәҗж”ҝзӯ–пјҢеҸӘжңүз”ЁжңҚеҠЎеҷЁеҹҹеҗҚи®ҫзҪ®зҡ„CookieжүҚдјҡдёҠдј пјҢе…¶д»–еҹҹеҗҚзҡ„Cookieе№¶дёҚдјҡдёҠдј пјҢдё”пјҲи·ЁжәҗпјүеҺҹзҪ‘йЎөд»Јз Ғдёӯзҡ„document.cookieд№ҹж— жі•иҜ»еҸ–жңҚеҠЎеҷЁеҹҹеҗҚдёӢзҡ„CookieгҖӮ

йқһз®ҖеҚ•иҜ·жұӮ

йў„жЈҖиҜ·жұӮ

йқһз®ҖеҚ•иҜ·жұӮжҳҜйӮЈз§ҚеҜ№жңҚеҠЎеҷЁжңүзү№ж®ҠиҰҒжұӮзҡ„иҜ·жұӮпјҢжҜ”еҰӮиҜ·жұӮж–№жі•жҳҜPUTжҲ–DELETEпјҢжҲ–иҖ…Content-Typeеӯ—ж®өзҡ„зұ»еһӢжҳҜapplication/jsonгҖӮ

йқһз®ҖеҚ•иҜ·жұӮзҡ„CORSиҜ·жұӮпјҢдјҡеңЁжӯЈејҸйҖҡдҝЎд№ӢеүҚпјҢеўһеҠ дёҖж¬ЎHTTPжҹҘиҜўиҜ·жұӮпјҢз§°дёәвҖқйў„жЈҖвҖқиҜ·жұӮпјҲpreflightпјүгҖӮ

жөҸи§ҲеҷЁе…ҲиҜўй—®жңҚеҠЎеҷЁпјҢеҪ“еүҚзҪ‘йЎөжүҖеңЁзҡ„еҹҹеҗҚжҳҜеҗҰеңЁжңҚеҠЎеҷЁзҡ„и®ёеҸҜеҗҚеҚ•д№ӢдёӯпјҢд»ҘеҸҠеҸҜд»ҘдҪҝз”Ёе“ӘдәӣHTTPеҠЁиҜҚе’ҢеӨҙдҝЎжҒҜеӯ—ж®өгҖӮеҸӘжңүеҫ—еҲ°иӮҜе®ҡзӯ”еӨҚпјҢжөҸи§ҲеҷЁжүҚдјҡеҸ‘еҮәжӯЈејҸзҡ„XMLHttpRequestиҜ·жұӮпјҢеҗҰеҲҷе°ұжҠҘй”ҷгҖӮ

дёҠйқўд»Јз ҒдёӯпјҢHTTPиҜ·жұӮзҡ„ж–№жі•жҳҜPUTпјҢе№¶дё”еҸ‘йҖҒдёҖдёӘиҮӘе®ҡд№үеӨҙдҝЎжҒҜX-Custom-HeaderгҖӮ
жөҸи§ҲеҷЁеҸ‘зҺ°пјҢиҝҷжҳҜдёҖдёӘйқһз®ҖеҚ•иҜ·жұӮпјҢе°ұиҮӘеҠЁеҸ‘еҮәдёҖдёӘвҖқйў„жЈҖвҖқиҜ·жұӮпјҢиҰҒжұӮжңҚеҠЎеҷЁзЎ®и®ӨеҸҜд»Ҙиҝҷж ·иҜ·жұӮгҖӮдёӢйқўжҳҜиҝҷдёӘвҖқйў„жЈҖвҖқиҜ·жұӮзҡ„HTTPеӨҙдҝЎжҒҜгҖӮ

вҖңйў„жЈҖвҖқиҜ·жұӮз”Ёзҡ„иҜ·жұӮж–№жі•жҳҜOPTIONSпјҢиЎЁзӨәиҝҷдёӘиҜ·жұӮжҳҜз”ЁжқҘиҜўй—®зҡ„гҖӮеӨҙдҝЎжҒҜйҮҢйқўпјҢе…ій”®еӯ—ж®өжҳҜOriginпјҢиЎЁзӨәиҜ·жұӮжқҘиҮӘе“ӘдёӘжәҗгҖӮ

йҷӨдәҶOriginеӯ—ж®өпјҢвҖқйў„жЈҖвҖқиҜ·жұӮзҡ„еӨҙдҝЎжҒҜеҢ…жӢ¬дёӨдёӘзү№ж®Ҡеӯ—ж®өгҖӮ

пјҲ1пјүAccess-Control-Request-Method

иҜҘеӯ—ж®өжҳҜеҝ…йЎ»зҡ„пјҢз”ЁжқҘеҲ—еҮәжөҸи§ҲеҷЁзҡ„CORSиҜ·жұӮдјҡз”ЁеҲ°е“ӘдәӣHTTPж–№жі•пјҢдёҠдҫӢжҳҜPUTгҖӮ

пјҲ2пјүAccess-Control-Request-Headers

иҜҘеӯ—ж®өжҳҜдёҖдёӘйҖ—еҸ·еҲҶйҡ”зҡ„еӯ—з¬ҰдёІпјҢжҢҮе®ҡжөҸи§ҲеҷЁCORSиҜ·жұӮдјҡйўқеӨ–еҸ‘йҖҒзҡ„еӨҙдҝЎжҒҜеӯ—ж®өпјҢдёҠдҫӢжҳҜX-Custom-HeaderгҖӮ

йў„жЈҖиҜ·жұӮзҡ„еӣһеә”

жңҚеҠЎеҷЁж”¶еҲ°вҖқйў„жЈҖвҖқиҜ·жұӮд»ҘеҗҺпјҢжЈҖжҹҘдәҶOriginгҖҒAccess-Control-Request-Methodе’ҢAccess-Control-Request-Headersеӯ—ж®өд»ҘеҗҺпјҢзЎ®и®Өе…Ғи®ёи·ЁжәҗиҜ·жұӮпјҢе°ұеҸҜд»ҘеҒҡеҮәеӣһеә”гҖӮ

дёҠйқўзҡ„HTTPеӣһеә”дёӯпјҢе…ій”®зҡ„жҳҜAccess-Control-Allow-Originеӯ—ж®өпјҢиЎЁзӨәhttp://api.bob.comеҸҜд»ҘиҜ·жұӮж•°жҚ®гҖӮиҜҘеӯ—ж®өд№ҹеҸҜд»Ҙи®ҫдёәжҳҹеҸ·пјҢиЎЁзӨәеҗҢж„Ҹд»»ж„Ҹи·ЁжәҗиҜ·жұӮгҖӮ

еҰӮжһңжөҸи§ҲеҷЁеҗҰе®ҡдәҶвҖқйў„жЈҖвҖқиҜ·жұӮпјҢдјҡиҝ”еӣһдёҖдёӘжӯЈеёёзҡ„HTTPеӣһеә”пјҢдҪҶжҳҜжІЎжңүд»»дҪ•CORSзӣёе…ізҡ„еӨҙдҝЎжҒҜеӯ—ж®өгҖӮиҝҷж—¶пјҢжөҸи§ҲеҷЁе°ұдјҡи®Өе®ҡпјҢжңҚеҠЎеҷЁдёҚеҗҢж„Ҹйў„жЈҖиҜ·жұӮпјҢеӣ жӯӨи§ҰеҸ‘дёҖдёӘй”ҷиҜҜпјҢиў«XMLHttpRequestеҜ№иұЎзҡ„onerrorеӣһи°ғеҮҪж•°жҚ•иҺ·гҖӮжҺ§еҲ¶еҸ°дјҡжү“еҚ°еҮәеҰӮдёӢзҡ„жҠҘй”ҷдҝЎжҒҜгҖӮ

жңҚеҠЎеҷЁеӣһеә”зҡ„е…¶д»–CORSзӣёе…іеӯ—ж®өеҰӮдёӢгҖӮ

пјҲ1пјүAccess-Control-Allow-Methods

иҜҘеӯ—ж®өеҝ…йңҖпјҢе®ғзҡ„еҖјжҳҜйҖ—еҸ·еҲҶйҡ”зҡ„дёҖдёӘеӯ—з¬ҰдёІпјҢиЎЁжҳҺжңҚеҠЎеҷЁж”ҜжҢҒзҡ„жүҖжңүи·ЁеҹҹиҜ·жұӮзҡ„ж–№жі•гҖӮжіЁж„ҸпјҢиҝ”еӣһзҡ„жҳҜжүҖжңүж”ҜжҢҒзҡ„ж–№жі•пјҢиҖҢдёҚеҚ•жҳҜжөҸи§ҲеҷЁиҜ·жұӮзҡ„йӮЈдёӘж–№жі•гҖӮиҝҷжҳҜдёәдәҶйҒҝе…ҚеӨҡж¬ЎвҖқйў„жЈҖвҖқиҜ·жұӮгҖӮ

пјҲ2пјүAccess-Control-Allow-Headers

еҰӮжһңжөҸи§ҲеҷЁиҜ·жұӮеҢ…жӢ¬Access-Control-Request-Headersеӯ—ж®өпјҢеҲҷAccess-Control-Allow-Headersеӯ—ж®өжҳҜеҝ…йңҖзҡ„гҖӮе®ғд№ҹжҳҜдёҖдёӘйҖ—еҸ·еҲҶйҡ”зҡ„еӯ—з¬ҰдёІпјҢиЎЁжҳҺжңҚеҠЎеҷЁж”ҜжҢҒзҡ„жүҖжңүеӨҙдҝЎжҒҜеӯ—ж®өпјҢдёҚйҷҗдәҺжөҸи§ҲеҷЁеңЁвҖқйў„жЈҖвҖқдёӯиҜ·жұӮзҡ„еӯ—ж®өгҖӮ

пјҲ3пјүAccess-Control-Allow-Credentials

иҜҘеӯ—ж®өдёҺз®ҖеҚ•иҜ·жұӮж—¶зҡ„еҗ«д№үзӣёеҗҢгҖӮ

пјҲ4пјүAccess-Control-Max-Age

иҜҘеӯ—ж®өеҸҜйҖүпјҢз”ЁжқҘжҢҮе®ҡжң¬ж¬Ўйў„жЈҖиҜ·жұӮзҡ„жңүж•ҲжңҹпјҢеҚ•дҪҚдёәз§’гҖӮдёҠйқўз»“жһңдёӯпјҢжңүж•ҲжңҹжҳҜ20еӨ©пјҲ1728000з§’пјүпјҢеҚіе…Ғи®ёзј“еӯҳиҜҘжқЎеӣһеә”1728000з§’пјҲеҚі20еӨ©пјүпјҢеңЁжӯӨжңҹй—ҙпјҢдёҚз”ЁеҸ‘еҮәеҸҰдёҖжқЎйў„жЈҖиҜ·жұӮгҖӮ

4.3 жөҸи§ҲеҷЁзҡ„жӯЈеёёиҜ·жұӮе’Ңеӣһеә”
дёҖж—ҰжңҚеҠЎеҷЁйҖҡиҝҮдәҶвҖқйў„жЈҖвҖқиҜ·жұӮпјҢд»ҘеҗҺжҜҸж¬ЎжөҸи§ҲеҷЁжӯЈеёёзҡ„CORSиҜ·жұӮпјҢе°ұйғҪи·ҹз®ҖеҚ•иҜ·жұӮдёҖж ·пјҢдјҡжңүдёҖдёӘOriginеӨҙдҝЎжҒҜеӯ—ж®өгҖӮжңҚеҠЎеҷЁзҡ„еӣһеә”пјҢд№ҹйғҪдјҡжңүдёҖдёӘAccess-Control-Allow-OriginеӨҙдҝЎжҒҜеӯ—ж®өгҖӮ

дёӢйқўжҳҜвҖқйў„жЈҖвҖқиҜ·жұӮд№ӢеҗҺпјҢжөҸи§ҲеҷЁзҡ„жӯЈеёёCORSиҜ·жұӮгҖӮ

дёҠйқўеӨҙдҝЎжҒҜзҡ„Originеӯ—ж®өжҳҜжөҸи§ҲеҷЁиҮӘеҠЁж·»еҠ зҡ„гҖӮ

дёӢйқўжҳҜжңҚеҠЎеҷЁжӯЈеёёзҡ„еӣһеә”гҖӮ

дёҠйқўеӨҙдҝЎжҒҜдёӯпјҢAccess-Control-Allow-Originеӯ—ж®өжҳҜжҜҸж¬Ўеӣһеә”йғҪеҝ…е®ҡеҢ…еҗ«зҡ„гҖӮ
